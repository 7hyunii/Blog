효율적이고 확장 가능한 REST API를 설계하기 위한 핵심 원칙들을 정리합니다.

## REST의 기본 원칙

REST는 Representational State Transfer의 약자로, **리소스 중심**의 API 설계 방법입니다.

### 1. 명사를 사용하라

URL은 동사가 아닌 명사로 구성합니다.

```
GET /users
POST /users
GET /users/123
PUT /users/123
DELETE /users/123
```

### 2. HTTP 메서드를 적절히 활용

- **GET**: 조회
- **POST**: 생성
- **PUT/PATCH**: 수정
- **DELETE**: 삭제

### 3. 상태 코드 활용

```
200 OK - 성공
201 Created - 생성 성공
400 Bad Request - 잘못된 요청
401 Unauthorized - 인증 필요
404 Not Found - 리소스 없음
500 Internal Server Error - 서버 오류
```

## 버전 관리

API는 반드시 버전을 명시해야 합니다.

```
/api/v1/users
/api/v2/users
```
