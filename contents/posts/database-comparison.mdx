프로젝트에 적합한 데이터베이스를 선택하기 위해 SQL과 NoSQL 데이터베이스의 특징을 비교합니다.

## SQL vs NoSQL 주요 차이점

| 구분 | SQL | NoSQL |
|-----|-----|-------|
| **데이터 모델** | 테이블 기반 (관계형) | 문서, Key-Value, 그래프 등 |
| **스키마** | 고정된 스키마 (정적) | 유연한 스키마 (동적) |
| **확장성** | 수직 확장 (Scale-up) | 수평 확장 (Scale-out) |
| **트랜잭션** | ACID 보장 | BASE 모델 (일부 예외) |
| **쿼리 언어** | SQL | 데이터베이스마다 다름 |
| **사용 사례** | 복잡한 쿼리, 관계형 데이터 | 대용량 데이터, 빠른 변화 |

## 주요 데이터베이스 비교

### SQL 데이터베이스

| 데이터베이스 | 특징 | 장점 | 단점 |
|-----------|------|------|------|
| **MySQL** | 가장 널리 사용 | 성숙한 생태계, 무료 | 복잡한 쿼리에서 성능 저하 |
| **PostgreSQL** | 오픈소스, 고급 기능 | JSON 지원, 확장성 우수 | 초기 설정 복잡 |
| **Oracle** | 엔터프라이즈급 | 강력한 성능, 안정성 | 높은 라이선스 비용 |
| **SQL Server** | Microsoft 제품 | Azure 통합 우수 | Windows 종속성 |

### NoSQL 데이터베이스

| 데이터베이스 | 타입 | 특징 | 적합한 용도 |
|-----------|------|------|-----------|
| **MongoDB** | Document | JSON 기반 문서 저장 | 컨텐츠 관리, 실시간 분석 |
| **Redis** | Key-Value | 인메모리 저장소 | 캐싱, 세션 관리 |
| **Cassandra** | Wide-Column | 고가용성, 분산 | 대용량 시계열 데이터 |
| **Neo4j** | Graph | 그래프 기반 | 소셜 네트워크, 추천 시스템 |

## 선택 가이드

### SQL 데이터베이스를 선택해야 할 때

```
✅ 데이터 구조가 명확하고 자주 변경되지 않을 때
✅ 복잡한 관계와 조인이 필요할 때
✅ ACID 트랜잭션이 중요할 때
✅ 데이터 무결성이 최우선일 때

예: 은행 시스템, ERP, 전자상거래
```

### NoSQL 데이터베이스를 선택해야 할 때

```
✅ 데이터 구조가 유동적일 때
✅ 대용량 데이터를 빠르게 처리해야 할 때
✅ 수평 확장이 필요할 때
✅ 비정형 데이터를 다룰 때

예: 소셜 미디어, IoT 데이터, 실시간 분석
```

## 실전 예시

### 전자상거래 프로젝트

```javascript
// SQL: 주문 데이터 (관계가 중요)
const orders = await db.query(`
  SELECT o.*, u.name, p.title
  FROM orders o
  JOIN users u ON o.user_id = u.id
  JOIN products p ON o.product_id = p.id
  WHERE o.status = 'pending'
`);

// NoSQL: 제품 카탈로그 (유연한 스키마)
const products = await mongodb.collection('products').find({
  category: 'electronics',
  'specs.brand': 'Samsung'
});
```
